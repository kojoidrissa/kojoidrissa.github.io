2014-12-08T12:27:41-06:00

Today, we play with FORMS!

The form element, <form> requires an action attribute and should have a method and id attribute.

The method attribute will be get or put. It defaults to get if no method is specified (although it's best to give one). The id attribute behaves like it does in other elements.

This is where the form controls will appear. Or, would, if we used an input element to provide them

Username:

Each form control (created by the input element) requires a name attribute, to identify the data being entered. The type attribute determines what KIND of form control is used. The size attribute is depricated, with CSS handling those duties now. maxlength is used to control the length of the data being entered.

The textarea element creates a multi-line area to enter text. This element needs an opening and closing tag, unlike other input elements, which are 'empty'. You should use CSS to control the size of a textarea control.

Various input types and their controls

These take the form <input type="TYPENAME" /> in your HTML.

  • text: single-line text field
  • password: as above, but with the entered characters hidden. They're only hidden from view, not secured or encrypted.
  • radio: allows users to choose 1 and ONLY 1 of multiple options
    • The name attribute names the type of question being asked. All questions of the same category should share the same name attribute.
    • The value attribute tells the server which answer has been selected. The value for every radio button should be unique w/in a group of questions
  • checkbox: allows users to choose more than one answer. The name and value attributes
    • function as with the radio type above.
